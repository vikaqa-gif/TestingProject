Тест-план - Petstore API

1. Цель тестирования
Проверить, что API корректно обрабатывает CRUD-операции для pet, store, user, включая обязательные
и опциональные параметры, и возвращает ожидаемые статус-коды и ответы.
Типы проверок: позитивные, негативные, граничные

2. Область тестирования
- CRUD-операции с питомцами pet
- Операции с заказами store, order оформление заказа, получение и удаление заказа, проверка инвентаря
- Операции с пользователями user, включая логин/логаут
- Проверка статус-кодов и обязательных полей
- Проверка позитивных, негативных и граничных сценариев.

3. Окружение:
- публичный Petstore https://petstore.swagger.io

 4. Стратегия тестовых данных
- Использовать уникальные id и username
- После выполнения тестов удалять созданные данные (чистка окружения)
- Все тесты должны быть независимыми друг от друга

 5. Приоритеты
- Высокий: Smoke + CRUD для pet, store, user
- Средний: граничные проверки 
- Низкий: негативные проверки

6. Риски и допущения
- Petstore - публичное демо, данные могут меняться другими пользователями
- Возможны расхождения между документацией и фактическим поведением
- Нет ограничений по количеству запросов, но возможны случайные ошибки 500  
- Отсутствие строгой валидации на сервере принимаются некорректные данные (id=-1)
